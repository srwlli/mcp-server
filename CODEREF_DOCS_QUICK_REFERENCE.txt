What Can coderef-docs Generate?

================================================================================
DOMAIN 1: FOUNDATION DOCUMENTS (8 Types Available)
================================================================================

Using: generate_foundation_docs (generates 4-5 at once)
   or: generate_individual_doc (generates 1 specific type)

Available Document Types:

1. README.md
   Purpose: Project overview, quick start, installation
   Audience: Everyone
   Content: Purpose, overview, features, installation, usage, contributing

2. ARCHITECTURE.md
   Purpose: System design, patterns, technical decisions
   Audience: Developers
   Content: Architecture diagram, design decisions, patterns used, layers

3. SCHEMA.md
   Purpose: Data models, database design, relationships
   Audience: Backend developers, DBAs
   Content: Entities, relationships, field types, constraints

4. API.md
   Purpose: API endpoints, request/response formats
   Audience: API consumers, frontend developers
   Content: Base URL, endpoints, parameters, examples, error codes

5. COMPONENTS.md
   Purpose: Component hierarchy, reusable parts
   Audience: Frontend developers
   Content: Component tree, props, usage examples, storybook links

6. USER-GUIDE.md
   Purpose: How to use the application
   Audience: End users
   Content: Features, workflows, troubleshooting, FAQs

7. MY-GUIDE.md
   Purpose: Custom reference guide (pick your topic)
   Audience: Varies
   Content: Whatever you define

8. FEATURES.md
   Purpose: Feature list, roadmap, status
   Audience: Product teams, users
   Content: Current features, planned features, deprecated features

All follow POWER Framework:
✓ Purpose (why it exists)
✓ Overview (what's included)
✓ What/Why/When (details)
✓ Examples (concrete)
✓ References (links)

Real Example: We generated all 5 core docs for 4 servers
         = 20 foundation documents proven working ✅

================================================================================
DOMAIN 2: QUICK REFERENCE GUIDES (5 App Types)
================================================================================

Using: generate_quickref_interactive

Purpose: 150-250 line scannable reference for any application

Supported App Types:

1. CLI (Command-Line Interface)
   Output: Command list, flags, options, usage examples
   Example: "coderef-context --scan /path/to/project"

2. WEB (Web Application)
   Output: Pages, routes, features, UI flows
   Example: Dashboard → Settings → Profile

3. API (REST/GraphQL API)
   Output: Endpoints, methods, parameters, examples
   Example: GET /api/v1/users/{id}

4. DESKTOP (Desktop Application)
   Output: Menu items, features, shortcuts, workflows
   Example: File → Open → Recent

5. LIBRARY (Code Library/Package)
   Output: Functions, classes, imports, examples
   Example: from coderef import scan; scan(path)

Process:
1. You specify app type
2. AI asks 10-15 clarifying questions
3. AI generates scannable quickref.md
4. Output is organized and professional

DOMAIN 3: STANDARDS & COMPLIANCE (3 Tools)
================================================================================

Tool 1: establish_standards
   Scans codebase and extracts patterns
   Discovers:
     ✓ UI component patterns (buttons, modals, forms)
     ✓ Behavior patterns (error handling, loading)
     ✓ UX flow patterns (navigation, auth, permissions)
   Output: 4 markdown files
   Depth options: quick (1-2 min), standard (3-5 min), deep (10-15 min)

Tool 2: audit_codebase
   Checks code against established standards
   Scores: 0-100 compliance
   Issues: Lists violations by severity
   Output: Audit report with fix suggestions

Tool 3: check_consistency
   Pre-commit quality gate
   Purpose: Prevent violations from being committed
   Works with: Git staged files
   Output: Pass/fail for CI/CD pipelines
   Thresholds: critical, major, or minor

================================================================================
CHANGELOG MANAGEMENT (3 Tools)
================================================================================

Tool 1: get_changelog
   Query changelog entries
   Filter by: version, change type, date range
   Output: Structured changelog entries

Tool 2: add_changelog_entry
   Manually add changelog entry
   Captures: version, type, title, description, files, reason, impact
   Output: Updates CHANGELOG.json

Tool 3: record_changes ⭐ (SMART)
   Auto-detect git changes
   Suggests: change type from commit message
   Calculates: Severity level
   Shows: Preview for confirmation
   Output: CHANGELOG.json + README version bump

   Advantage: Agentic = AI reviews changes automatically

COMMON WORKFLOWS
================================================================================

WORKFLOW 1: Generate Project Docs
  Command: /generate-docs
  Input: Project path
  Output: README.md, ARCHITECTURE.md, SCHEMA.md, API.md, COMPONENTS.md
  Time: ~2 minutes

WORKFLOW 2: Create CLI Reference
  Command: /generate-quickref
  Select: CLI app type
  Output: 200-line quickref.md with commands, options, examples
  Time: ~5 minutes (includes AI interview)

WORKFLOW 3: Record Feature Completion
  Command: /record-changes
  Process: Git auto-detect → AI suggests → User confirms
  Output: CHANGELOG.json updated, README version bumped
  Time: ~1 minute

WORKFLOW 4: Establish Standards
  Command: /establish-standards
  Process: Scan codebase, extract patterns
  Output: 4 standards markdown files
  Time: 1-15 minutes (depends on depth)

WORKFLOW 5: Quality Check
  Command: /check-consistency
  Process: Check staged files against standards
  Result: Pass/fail for commit
  Time: ~10 seconds
